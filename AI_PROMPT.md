# AI 提示词

如果你想让 AI 助手（如 Claude、ChatGPT）帮你生成或修改 Clash Verge 扩展脚本，可以使用以下提示词：

---

## 提示词模板

```
请你扮演一位精通 Clash 配置的专家。我需要你帮我修改并重新编写一份完整的 Clash Verge 的扩展脚本。

我的需求如下，请严格执行：

1. **添加静态IP节点**：

依据我提供的以下信息，添加一个 Socks5 类型的节点：

* IP: [填写你的静态IP]
* 端口: [填写端口]
* 用户名: [填写用户名]
* 密码: [填写密码]
* 名称: [填写节点名称，如：🇺🇸 静态IP-洛杉矶]

2. **创建自动优选入口（前置节点）**：

* 创建一个名为 `⚡️ 自动优选 (入口)` 的策略组。
* 类型必须是 `url-test`（自动测速），间隔 `300` 秒。
* 请从我下方提供的配置文件中，提取所有正常的机场节点（如香港、美国、日本、新加坡等）放入这个组，让它自动选择延迟最低的节点。

3. **创建链式代理（核心需求）**：

* 创建一个名为 `🔗 静态IP链式 (最终)` 的策略组。
* 类型必须是 `select`（手动选择）。
* 使用 `dialer-proxy` 实现链式代理。
* 流量路径：`你的设备` -> `⚡️ 自动优选 (入口)` -> `静态IP节点` -> `目标网站`

4. **整合策略组**：

* 将这个 `🔗 静态IP链式 (最终)` 组，加入到主要的策略组选项中（如 OpenAI, Google, TikTok, 以及主漏网之鱼 Proxy 组），方便我手动切换。

5. **数据来源**：

* 基于我当前使用的 Clash 订阅配置
* 脚本需要在订阅更新后自动执行，动态添加配置
```

---

## 使用说明

1. **复制上面的提示词**
2. **填写你的静态IP信息**（IP、端口、用户名、密码、节点名称）
3. **发送给 AI 助手**（如 Claude、ChatGPT）
4. **AI 会生成完整的扩展脚本**
5. **将生成的脚本复制到 Clash Verge 的扩展脚本中**

## 注意事项

- 确保提供的静态IP信息准确无误
- 如果使用其他协议（如 HTTP、Trojan），需要在提示词中说明
- 可以根据实际需求调整策略组名称和测速间隔
- 生成的脚本可以根据需要进一步自定义
